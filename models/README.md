# models/ — 랭커 학습/추론 & 벤치마크 (레이어 C의 NN 부분)

추천 점수 중 **학습 랭커(NN)** 담당. 도달성/시간 계산은 여기 없음(그건 `backend/`의 결정론 엔진).
**NN은 이 과제의 주인공** — 직접 구축·학습하는 것이 핵심 학습 목표.
> ⚠️ 아직 코드 미작성. NN 트랙은 Colab에서 v1.0과 **병렬로 일찍 시작 가능**(웹앱 완성 대기 불필요). 앱 통합은 v1.4.

## 역할

- **NN은 후보 랭킹만** 한다 (시간/도달성 아님).
- Foursquare 공개 체크인으로 *방법을 학습·검증*(정량지표) → 그 구조를 종로 POI에 적용(데모).
- 각 레벨에서 **모델 정의 → 학습 루프 → loss 관찰 → 평가**를 직접 손으로.

## ★ NN 사다리 (L1~L4, 직접 구축·학습)

| Level | 모델 | 직접 경험하는 것 | 산출/근거 |
|---|---|---|---|
| **L1** | MLP 랭커 (`nn.Linear`+ReLU → 점수) | 모델 정의·순전파·손실·옵티마이저·학습 루프 | NN 입문 완주, Recall@k/NDCG |
| **L2** | 학습된 임베딩 (`nn.Embedding`) | 표현 학습, UMAP로 임베딩 시각화 | POI가 종류별로 뭉치는 그림 |
| **L3** | 시퀀스 모델 (LSTM / 소형 Transformer) | A→B→C를 *순서*로 인코딩, 다음 장소 예측 | "층층이 펼쳐지는 NN" 실구현 |
| **L4** *(선택)* | GNN (POI 그래프 메시지 패싱) | 그래프 표현학습 | 고급 확장 |
| **v2** *(연기)* | 강화학습 정책 (MDP) | 전체 동선 보상 최적화 | future work |

> v1 범위는 **L1~L3 + 베이스라인 비교**. L4(GNN)는 여유 시, RL은 v2로 연기.

## 베이스라인 비교 (과학적 서사)

인기순 · KNN · 로지스틱 · 결정트리 · SVM · 앙상블을 비교군으로 두고
**위 NN 사다리와 Recall@k · NDCG로 대결** → "고전 대비 NN 우위"를 수치로 증명.
(= NN이 장식이 아니라 *증명된 선택*)

## 표현/전처리 기법 (보조)

DBSCAN(핫스팟 군집) · PCA/UMAP(차원축소·시각화) · 판별분석/선형회귀(보조 분류·예측).

## 서울 적용 (데모)

- 학습된 구조를 종로 POI에 적용.
- 전이 신호: 데이터랩 "관광지별 연관 관광지" + 부족분은 합성 궤적.
- **정직한 한계:** Foursquare→종로 도메인 갭. 사용자에게 보이는 순위는 해석가능 신호(언급량·군집·연관관광지)가 주로 끌고, 학습랭커는 벤치마크 증거 역할.
